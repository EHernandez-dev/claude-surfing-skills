# EOT20 as a free, keyless, offline tide rung below WorldTides

After NOAA (US-only, ADR 0001) and the optional paid WorldTides key, non-US spots with no key still fell back to a manual note, so the whole Spanish coast (every home spot) rendered with no tide curve and, since the hourly conditions strip hangs off the tide chart, no strip either. Windguru and similar sites show tides everywhere because they do not own tide data: they compute it from a global tidal-harmonic model. We do the same. We add the **EOT20** global harmonic model (Hart-Davis et al.; SEANOE DOI 10.17882/79489) as a fourth tide rung, predicted locally with `pyTMD` (no API key, no network at prediction time). The ladder is now NOAA → WorldTides (if key) → EOT20 (if installed) → manual note. `fetch_tides_harmonic` samples the elevation every 10 min over the forecast window and reduces it to high/low extremes (`find_tide_extremes`); the output reuses the existing tides contract shape, so nothing downstream (`render_report.py`, the report template) changed, and once extremes exist the hourly strip renders for these spots for free.

EOT20 is an **optional** install, not a base dependency: `pyTMD` pulls a scientific stack (numpy/scipy/pyproj/xarray/...) and the model is a one-time ~2 GB download (`download_tide_model.py` into `EOT20_DIR`). Absent the dependency or the files, or for an off-grid point, the rung returns the standard gap shape and the tool behaves exactly as before (exit-0 contract, lightweight zero-config default both preserved). Coastal prediction uses `extrapolate=True`: surf spots sit on the coast and rivermouths (Plentzia) that the 1/8° grid may mask, so extrapolation to the nearest ocean cell is preferred over returning NaN, trading a little accuracy for a usable curve. Heights are labelled datum **MSL**.

## Considered Options

- **Paid WorldTides for everyone / a WorldTides key as the answer**: rejected as the default. The free tier is 100 credits then paid; the user's hard constraint was zero cost. WorldTides stays *above* EOT20 in the ladder because setting a key is an explicit opt-in to station-grade extremes on chart datum, which read closer to printed local tables (ADR 0001's reasoning) than EOT20's mean-sea-level harmonic.
- **Scraping Windguru/Surfline for the tide**: rejected. Their ToS forbid automated access (ADR 0003), and the tide they display is itself a derived harmonic value whose source (a global model) is directly and freely available to us. Screen-scraping a derived number through a ToS violation is strictly worse than computing from the source.
- **FES2022 or NASA GOT instead of EOT20**: rejected. FES2022 needs AVISO registration and is larger; GOT carries fewer constituents. EOT20 is open CC-BY 4.0 with no registration, `pyTMD`-supported, and accurate enough for surf timing.
- **MSL heights** (which ADR 0001 rejected for WorldTides): accepted here, because a harmonic model has no chart-datum option and the alternative is no tide at all. The reliable signal for surf is the *timing* of highs and lows, not absolute height; the report says so and flags reduced estuary accuracy. Chart-datum sources still win when available, which is why EOT20 sits at the bottom of the ladder.
- **Bundling the model in the repo / auto-downloading on first run**: rejected. 2 GB does not belong in git, and a surprise multi-hundred-MB download mid-report is hostile; setup is an explicit one-time command.
